---

## ✅ 问题根源

**字段名不匹配**：
- **后端 API**: 返回 `time` 字段（格式："2026-02-05 09:27"）
- **前端代码**: 尝试访问 `date` 字段
- **结果**: `dates` 数组全部为 `undefined`，导致图表无法绘制

---

## 🔧 修复方案

### 修改文件
`/home/tzj/project/python/PythonProject/templates/index.html`

### 修改内容
```javascript
// 修复前（第 433 行）
const dates = historyData.map(d => d.date);

// 修复后
// 兼容 time 和 date 两种字段
const dates = historyData.map(d => d.time || d.date || '未知日期');
```

### 修复优势
1. ✅ **向后兼容**: 同时支持 `time` 和 `date` 字段
2. ✅ **容错处理**: 如果两个字段都没有，返回 '未知日期' 而不是报错
3. ✅ **简洁优雅**: 一行代码解决问题

## 📝 知识点总结

### 1. 前后端数据契约
- ⚠️ **重要**: 前后端必须约定好字段名称
- ✅ **最佳实践**: 使用 TypeScript 或 OpenAPI 规范定义接口

### 2. 容错处理
```javascript
// 推荐方式：提供默认值
d.time || d.date || '未知日期'

// 不推荐：直接访问可能不存在的字段
d.date  // 可能返回 undefined
```

### 3. 调试技巧
当页面数据显示异常时：
1. 检查后端 API 返回的数据（浏览器 DevTools → Network）
2. 检查前端 JavaScript 如何处理数据（Console 打印）
3. 检查字段名是否匹配
