# 历史数据波动模拟脚本

## 📋 功能说明

为 Redis 中的物品历史价格数据添加**随机波动**，使图表显示更加真实、可观。

### 解决的问题
- ❌ **问题**: 当前所有物品的历史价格都是固定值，图表显示为直线
- ✅ **解决**: 生成 30 天的随机波动价格数据，让曲线更自然

## ⚙️ 参数说明

### 波动率 (Volatility)

| 波动率 | 效果 | 适用场景 |
|--------|------|----------|
| 0.05 (5%)  | 小幅波动，曲线平缓 | 稳定物品 |
| 0.15 (15%) | 中等波动，自然真实 | **推荐（默认）** |
| 0.30 (30%) | 大幅波动，曲线陡峭 | 高风险物品 |

### 历史天数

| 天数 | 效果 | 建议 |
|------|------|------|
| 7    | 一周数据 | 短期趋势 |
| 30   | 一个月数据 | **推荐（默认）** |
| 60   | 两个月数据 | 长期趋势 |

## 📊 算法原理

### 价格生成逻辑

```python
# 从今天往前倒推 30 天
for i in range(30):
    if i == 0:
        price = current_price  # 今天 = 当前价格
    else:
        # 随机波动 -15% ~ +15%
        change_rate = random.uniform(-0.15, 0.15)
        price = current / (1 + change_rate)
        current = price
```

### 示例

假设当前价格为 **1000 哈弗币**：

```
第 30 天（过去）: 950
第 29 天：980
第 28 天：920
...
第 2 天：1050
第 1 天：980
第 0 天（今天）: 1000  ← 保持当前价格
```

## ⚠️ 注意事项

### 1. **数据安全**
- ✅ 不会覆盖当前价格（保持原值）
- ✅ 只会添加新的历史记录
- ✅ 如果已有 >5 条历史记录，会自动跳过

### 2. **Redis 过期策略**
- 历史记录设置 **30 天过期**
- 与物品详情保持一致
- 定期清理避免内存占用

### 3. **性能影响**
- 单个物品：~30 条记录 × 493 个物品 ≈ **14,790 条记录**
- 执行时间：约 **10-20 秒**（取决于 Redis 性能）
- 内存占用：约 **1-2 MB**

2. **检查历史记录**:

