### 3. 根本原因

通过调试发现两个关键问题：

#### 问题 1: `datetime` 未导入
```python
# ❌ 错误代码（缺少导入）
def get_price_history(self, item_name: str, days: int = 7) -> List[Dict]:
    # ...
    for timestamp_str, price in history_data:
        timestamp = int(float(timestamp_str))
        result.append({
            'time': datetime.fromtimestamp(timestamp).strftime('%Y-%m-%d %H:%M'),  # ← NameError: name 'datetime' is not defined
            'price': price,
            'timestamp': timestamp
        })
```

**错误日志**:
```
ERROR:app.services.data_service:时间戳转换失败：1770254830, name 'datetime' is not defined
ERROR:app.services.data_service:时间戳转换失败：1770600430, name 'datetime' is not defined
... (35 条错误)
```

#### 问题 2: 时间范围过滤逻辑错误
即使修复了 `datetime` 导入，由于原始代码使用 `zrangebyscore` 方法，该方法要求 Sorted Set 的 score 是数值类型。但实际存储时使用的是字符串类型的时间戳 `str(timestamp)`，导致时间范围计算失效。

### 修复 2: 重写 get_price_history 方法

**问题**: 原代码使用 `zrangebyscore` 无法正确处理字符串类型的 score

**解决方案**: 改用 `zrange` 获取所有数据，然后在 Python 中进行时间过滤

```python
# 修复前
def get_price_history(self, item_name: str, days: int = 7) -> List[Dict]:
    history_key = f"trading:history:{item_name}"
    now = int(time.time())
    start_time = now - (days * 24 * 60 * 60)
    
    # ❌ 问题：zrangebyscore 要求 score 是数值，但我们存储的是字符串
    history_data = self.redis_client.zrangebyscore(
        history_key,
        min=start_time,
        max=now,
        withscores=True
    )
    # ... 转换逻辑

# 修复后
def get_price_history(self, item_name: str, days: int = 7) -> List[Dict]:
    history_key = f"trading:history:{item_name}"
    
    if not self.redis_client.exists(history_key):
        return []
    
    # ✅ 使用 zrange 获取所有数据（不依赖 score 类型）
    history_data = self.redis_client.zrange(history_key, 0, -1, withscores=True)
    
    if not history_data:
        return []
    
    # 转换为前端可用格式
    result = []
    for timestamp_str, price in history_data:
        try:
            # 将字符串时间戳转换为整数
            timestamp = int(float(timestamp_str))
            result.append({
                'time': datetime.fromtimestamp(timestamp).strftime('%Y-%m-%d %H:%M'),
                'price': price,
                'timestamp': timestamp
            })
        except Exception as e:
            logger.error(f"时间戳转换失败：{timestamp_str}, {e}")
            continue
    
    # 按时间排序（从旧到新）
    result.sort(key=lambda x: x['timestamp'])
    
    # 在 Python 中进行时间过滤
    if days and days > 0:
        now = int(time.time())
        start_time = now - (days * 24 * 60 * 60)
        result = [r for r in result if r['timestamp'] >= start_time]
    
    return result
```

## 💡 经验教训

### 1. 数据类型一致性
- ⚠️ **问题**: Redis Sorted Set 的 score 使用字符串存储，但用数值方法查询
- ✅ **解决**: 统一使用 `zrange` 获取所有数据，在应用层处理

### 2. 导入语句完整性
- ⚠️ **问题**: 使用了 `datetime` 但未导入
- ✅ **解决**: 检查所有使用的模块是否都已导入

### 3. 调试日志的重要性
- ⚠️ **问题**: 35 条转换错误但未被及时发现
- ✅ **解决**: 添加详细的 DEBUG 日志记录数据流转过程

### 4. 前后端字段兼容性
- ⚠️ **问题**: 后端返回 `time`，前端期望 `date`
- ✅ **解决**: 使用 `d.time || d.date` 兼容两种字段
